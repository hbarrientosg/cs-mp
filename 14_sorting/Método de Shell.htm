9. Método de Shell

9.1 Método

    El método de Shell es una extensión del de inserción lineal con intercambio, de tal forma que en el último paso de Shell coincide con éste. Los pasos precedentes usan la misma técnica, pero en vez de comparar e intercambiar elementos consecutivos, los compara tomando una cierta distancia. Por ejemplo, en el caso de que la distancia sea 4, la posición 1 se compara con la 5, la 5 con la 9, etc. A este tipo de comparaciones "hacia delante" se les denomina primarias. Cuando se verifica un intercambio se producen una serie de comparaciones "hacia atrás" denominadas secundarias, que implican a aquellos elementos que tomaron parte en la secuencia de comparaciones primarias. Así, cuando se produce un intercambio entre las posiciones 9 y 13 se produce una secuencia secundaria de comparaciones e intercambios entre las posiciones 5:9 y 1:5. Estas comparaciones secundarias se propagarán hasta que no exista intercambio o se alcance el principio de la lista. 

    La secuencia de las distancias usadas no es sagrada. Se puede usar cualquier secuencia h1 ,h2 , ..., ht siempre que el último incremento sea igual a 1 (ht=1). 

    Una lista de longitud n se divide en n/2 particiones ((n-1)/2 si el número es impar) cada una con dos elementos. Si n es impar la primera de las particiones contendrá tres elementos. La distancia entre los elementos de la partición es, según Shell, n/2 en el primer paso y en pasos posteriores se determina reduciendo la distancia actual a su mitad.

9.2 Algoritmo

procedimiento shell(vec,n)  
dist <- n/2
mientras dist > 0 hacer
   para i desde 1 hasta n-dist hacer  
      j <- i+dist
      mientras (j > dist) y (vec[j] < vec[j-dist]) hacer
         vec[j] <-> vec[j-dist]
         j <- j-dist
      fin mientras
   fin para
   dist <- dist/2
fin mientras
retornar

9.4 Discusión

    Existe una gran diferencia entre este método y el de inserción lineal con intercambio. El número de comparaciones es sensiblemente menor. Un ejemplo donde se aprecia esta diferencia es el caso de una lista de 20 elementos en orden inverso, las comparaciones realizadas por el método de Shell son 79, y por el otro 190. 

    Nótese que para el método de Shell el peor caso no se produce cuando la lista inicial esta en orden inverso. 

    Al iniciar el paso final de la versión de Shell existen dos particiones, la de las posiciones impares y la de las pares. Cada partición esta ordenada individualmente, pero no existe un orden relativo significativo entre ellas. Esta condición ocurrirá siempre que n sea potencia de 2, ya que los valores de las distancias serán siempre pares. No existirán comparaciones entre elementos de posiciones par e impar respectivamente. Como consecuencia de ello, el paso final habrá de fundir dos listas independientes. 

    El número esperado de comparaciones no ha sido determinado. Knuth en Sorting and Searching presenta un análisis del método en función de la secuencia de distancias usadas, para la secuencia empleada por Shell el algoritmo es de orden O(n1,5).
